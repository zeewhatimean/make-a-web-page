The World Wide Web

Invented in the early 1990's, the world wide web is a collection of HTML documents. HTML stands for Hyper Text Markup Language. It is the basis for almost every webpage. The links between pages are hyperlinks or links for short. They make the web "web-like".

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML is made up of:

  • Text Content: what you see
  • Markup: what it looks like (how it's arranged)
  • Reference to other Documents: images and videos
  • Links to other pages
HTML Markup

Markup is made up of "tags", it includes (1) a less than symbol, (2) the tag name, (3) the greater than symbol, (4) the contents, which can be other tags or just simply text, and (5) another less than symbol with a slash and tag name, ending with a greater than symbol.

Names are always the same, the only difference is the closing tag has a slash in front of the name.

The whole thing can also be referred to as an element.



Tags and Elements

Different tags can influence the effect of the text, for example, the b-tag makes anything between the opening tag and closing tag appear bold. The em tag makes text italized. If you forget to close the tag, everything after the opening tag will follow the instructions given by the first tag. That is why we sometimes refer to computers as being "stupid", due to their literal interpretations of all our commands.

The image tag allows you to include images.

**Images do not have a closing tag because there are no contents to an image. It is known as a void tag.

There are two different ways of making a break. The br tag is inline, where the p tag is block. The block makes an invisible box around the text, and you can adjust the height and width.

DOM
When programmers talk about the DOM, Document Object Model, they are talking about the tree-like structure of a page. DOM is a standard convention for representing and interacting with elements in HTML.

Everything Is a Box

Thinking of the things in terms of boxes makes things a lot easier. Websites have many different elements, such as images, text, and headers. By thinking of things in terms of boxes we can actually easily rearrange things if you decide to give users a different experience.

Everything on a webpage is made up of boxes the tag that allows us to do this is the div tag. This tag is a block tag the inline version of this is the span tag.

Lesson 3: Adding CSS Style and HTML Structure

Avoiding repetition

CSS, cascading style sheets, is a list of rules on the style of the document where the most specific rule applies. The most important thing to remember when designing the CSS sheet is to avoid repetition to save time and reduce chance of errors. The code that do the same thing will often repeat over and over, like copying and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times when experiencing repetition. This is why consistency is a key to a well-organized web page.

Inheritance and CSS

Inheritance, the idea that styling not only applies to the specified element but all its decendants unless over-ruled by a more specific rule, has many benefits to it; including saving bandwidth,enhancing loading time and saving time coding. However it is important to note that descendant elements do not inherit all properties in general it is only text related ones. This page goes in depth on the inheritance of CSS.

Attributes in CSS

You can set the style of each element by assigning attributes to tags or classes in CSS for different tags or classes
